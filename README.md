# Дэп-Чат

Реальный чат для общения с друзьями, похожий на Telegram. Вы можете зарегистрироваться, войти в систему и общаться в реальном времени. Просто поделитесь ссылкой с друзьями!

## Особенности

- 🔐 **Безопасная аутентификация** с JWT токенами и хешированием паролей
- ⚡ **Реальное время** через WebSocket (Socket.IO)
- 👥 **Список пользователей** онлайн и всех зарегистрированных
- 💬 **История сообщений** с сохранением в базе данных
- 📱 **Адаптивный дизайн** для мобильных устройств
- 🎨 **Современный интерфейс** с темной/светлой темой

## Технологии

- **Backend**: Node.js, Express.js
- **База данных**: SQLite
- **Реальное время**: Socket.IO
- **Аутентификация**: JWT, bcryptjs
- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Стили**: Flexbox, CSS Grid, адаптивный дизайн

## Установка и запуск

### 1. Клонирование и настройка

```bash
# Перейдите в папку проекта
cd dep-chat

# Установите зависимости
npm install
```

### 2. Создание демо-пользователя (опционально)

```bash
node init-demo.js
```

Будет создан пользователь:
- **Логин**: `demo`
- **Пароль**: `demo123`

### 3. Запуск сервера

```bash
# Запуск в режиме разработки (с nodemon)
npm run dev

# Или запуск напрямую
node server.js
```

Сервер запустится на порту 3000 (или на порту, указанном в переменной окружения PORT).

### 4. Откройте в браузере

Перейдите по адресу: [http://localhost:3000](http://localhost:3000)

## Использование

### Регистрация
1. На главной странице нажмите "Регистрация"
2. Введите имя пользователя (минимум 3 символа)
3. Введите пароль (минимум 6 символов)
4. Подтвердите пароль
5. Нажмите "Зарегистрироваться"

### Вход
1. Введите имя пользователя и пароль
2. Или используйте демо-учетные данные:
   - **Логин**: `demo`
   - **Пароль**: `demo123`

### Чат
После входа вы попадете в интерфейс чата:
- **Левая панель**: список пользователей онлайн и всех пользователей
- **Основная область**: история сообщений
- **Нижняя панель**: поле для ввода сообщений

### Приглашение друзей
Нажмите кнопку "Пригласить" в чате и отправьте ссылку друзьям:
```
http://localhost:3000
```

## API Endpoints

### Аутентификация
- `POST /api/register` - регистрация нового пользователя
- `POST /api/login` - вход в систему
- `GET /api/me` - информация о текущем пользователе (требуется токен)

### Пользователи
- `GET /api/users` - список всех пользователей (требуется токен)

### Сообщения
- `GET /api/messages` - последние сообщения (требуется токен)

### WebSocket события
- `authenticate` - аутентификация через токен
- `sendMessage` - отправка сообщения
- `newMessage` - получение нового сообщения
- `userJoined` - пользователь присоединился
- `userLeft` - пользователь покинул чат
- `onlineUsers` - обновление списка онлайн пользователей

## Развертывание на хостинге

### 1. Подготовка к продакшену

1. Измените JWT секрет в `server.js`:
   ```javascript
   const JWT_SECRET = 'ваш-секретный-ключ-для-продакшена';
   ```

2. Настройте переменные окружения:
   ```bash
   export PORT=8080
   export JWT_SECRET=ваш-секретный-ключ
   ```

### 2. Использование PM2 для управления процессом

```bash
# Установите PM2 глобально
npm install -g pm2

# Запустите приложение
pm2 start server.js --name "dep-chat"

# Автозапуск при перезагрузке
pm2 startup
pm2 save
```

### 3. Использование Nginx как обратного прокси

Добавьте конфигурацию в `/etc/nginx/sites-available/dep-chat`:

```nginx
server {
    listen 80;
    server_name ваш-домен.ru;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## Безопасность

- Пароли хранятся в хешированном виде (bcrypt)
- Используются JWT токены с ограниченным временем жизни
- Защита от XSS через экранирование HTML
- CORS настроен для всех источников (в разработке)
- SQL-инъекции предотвращаются параметризованными запросами
