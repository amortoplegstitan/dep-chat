# Деплой Дэп-Чата на Railway.app

Пошаговая инструкция по развертыванию чата на Railway (исправленная версия, которая не крашится).

## Шаг 1: Подготовка проекта

Убедитесь, что в вашем проекте есть все необходимые файлы:
- `package.json` (с обновленными скриптами)
- `server.js` (исправленная версия)
- `src/database-railway.js` (база данных для Railway)
- `railway.json` (конфигурация Railway)
- Папка `public/` с фронтендом

## Шаг 2: Загрузите проект на GitHub (если еще не сделали)

1. Создайте репозиторий на GitHub
2. Загрузите все файлы через веб-интерфейс или Git
3. Убедитесь, что `.gitignore` исключает `node_modules/` и `chat.db`

## Шаг 3: Создайте проект на Railway

1. **Войдите на [railway.app](https://railway.app)**
2. Нажмите "New Project"
3. Выберите "Deploy from GitHub repo"
4. Найдите ваш репозиторий `dep-chat` и выберите его
5. Railway автоматически определит Node.js проект и начнет деплой

## Шаг 4: Настройте переменные окружения

После создания проекта:

1. Перейдите в настройки проекта (Settings)
2. Найдите раздел "Variables"
3. Добавьте следующие переменные:

| Переменная | Значение | Обязательно |
|------------|----------|-------------|
| `NODE_ENV` | `production` | Да |
| `JWT_SECRET` | Сгенерируйте случайную строку (минимум 32 символа) | Да |
| `PORT` | `3000` | Нет (Railway сам устанавливает) |
| `FRONTEND_URL` | URL вашего приложения на Railway (после деплоя) | Нет |

**Как сгенерировать JWT_SECRET:**
- Можно использовать команду: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
- Или сгенерировать на [randomkeygen.com](https://randomkeygen.com/)

## Шаг 5: Дождитесь деплоя

Railway автоматически:
- Установит зависимости (`npm install`)
- Запустит сервер (`npm start`)
- Назначит публичный URL вашему приложению

## Шаг 6: Проверьте работоспособность

1. После деплоя Railway покажет URL вашего приложения (например, `https://dep-chat.up.railway.app`)
2. Откройте этот URL в браузере
3. Проверьте health check: `https://ваш-проект.up.railway.app/health`
4. Должен появиться интерфейс регистрации/входа

## Шаг 7: Создайте первого пользователя

1. Нажмите "Регистрация" на главной странице
2. Создайте учетную запись
3. Или используйте демо-пользователя (если настроен)

## Возможные проблемы и решения

### Проблема 1: Сервер крашится сразу после запуска
**Решение:** Проверьте логи в Railway:
1. В панели Railway нажмите на ваш проект
2. Выберите вкладку "Logs"
3. Ищите ошибки (обычно это проблемы с базой данных или портом)

### Проблема 2: База данных не работает
**Решение:** Railway использует read-only файловую систему. Мы настроили SQLite на запись в `/tmp`:
- Проверьте, что используется `src/database-railway.js`
- Убедитесь, что есть права на запись в `/tmp`

### Проблема 3: WebSocket не работает
**Решение:** Railway поддерживает WebSocket. Убедитесь, что:
- В `server.js` правильно настроен Socket.IO
- В браузере нет ошибок CORS

### Проблема 4: Приложение работает, но друг не может подключиться
**Решение:** 
1. Убедитесь, что Railway дал публичный URL
2. Проверьте, что порт открыт (Railway делает это автоматически)
3. Отправьте другу полный URL (например, `https://dep-chat.up.railway.app`)

## Дополнительные настройки

### Настройка домена (опционально)
1. В Railway перейдите в Settings → Domains
2. Добавьте свой домен (если есть)
3. Настройте DNS записи как указано в инструкции

### Автоматическое обновление
Railway автоматически обновляет приложение при пуше в GitHub. Чтобы отключить:
- Settings → General → Disable "Auto Deploy"

### Мониторинг
Railway предоставляет:
- Логи в реальном времени
- Мониторинг использования ресурсов
- Метрики производительности

## Тестирование перед деплоем

Проверьте локально, что все работает:
```bash
# Установите зависимости
npm install

# Запустите сервер в production режиме
NODE_ENV=production JWT_SECRET=ваш-секрет node server.js

# Или используйте скрипт для Railway
npm run railway
```

## Контакты и поддержка

Если возникли проблемы:
1. Проверьте логи в Railway
2. Убедитесь, что все файлы загружены правильно
3. Сравните с рабочей версией: [Пример рабочего чата](https://github.com/ваш-логин/dep-chat)

## Важные замечания

1. **База данных SQLite в Railway** - данные могут быть потеряны при перезапуске контейнера. Для продакшена лучше использовать PostgreSQL.
2. **Бесплатный план Railway** имеет ограничения: 500 часов в месяц, 1 ГБ RAM.
3. **JWT_SECRET** должен быть сложным и храниться в секрете.
4. **Рекомендуется** включить HTTPS (Railway делает это автоматически).
